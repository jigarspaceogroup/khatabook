# Seed Data Documentation

## Overview

This document describes the test data generated by `prisma/seed.ts` for development and testing.

## Data Generated

### 1. Users (2)

- **User 1 & 2**: Indian names with realistic phone numbers and emails
- All users have `phone_verified: true` for easy testing
- Language code: `en` (English)
- Last login: Current timestamp

**Example:**
```typescript
{
  name: "Raj Sharma",
  phone_number: "+919876543210",
  email: "raj.sharma@example.com",
  phone_verified: true,
  language_code: "en"
}
```

### 2. Khatabooks (4 total - 2 per user)

Each user gets:
1. **Business Khatabook** (default)
   - Realistic Indian business name (e.g., "Sharma Kirana Store")
   - Business type extracted from name
   - `is_default: true`

2. **Personal Khatabook**
   - Name: "Personal Ledger"
   - `is_default: false`

### 3. Customers (40-60 total - 10-15 per khatabook)

- **Name**: Realistic Indian names (first + last)
- **Phone**: 70% have phone numbers (+91XXXXXXXXXX format)
- **Email**: 30% have email addresses
- **GSTIN**: 20% have GST identification numbers
- **Opening Balance**: Random between -₹500 and ₹1,000 (in paise)
- **Current Balance**: Calculated from opening_balance + transactions
- **Address**: Random Indian addresses
- **Notes**: 30% have notes

**Balance Distribution:**
- Mix of positive (customers owe you) and negative (you owe them)
- Realistic amounts based on transaction history

### 4. Transactions (100-200 total - 25-50 per khatabook)

- **Type**: 50% GAVE (credit), 50% GOT (debit)
- **Amount**: ₹100 to ₹50,000 in paise
  - 60% common amounts (₹100, ₹500, ₹1000, ₹2000, ₹5000, ₹10000)
  - 40% random amounts
- **Date**: Distributed over last 90 days
- **Payment Mode**: 25% each - CASH, UPI, CARD, null
- **Notes**: 40% have transaction notes

**Balance Calculation:**
```
current_balance = opening_balance + sum(GOT) - sum(GAVE)
```

### 5. Invoice Settings (4 - 1 per khatabook)

- **Business Details**: Name, address, phone, email, GSTIN
- **Invoice Prefix**: "INV"
- **Next Invoice Number**: 1
- **Bank Details**: JSON with account info
- **Terms**: Default payment terms (30 days)

### 6. Inventory Items (20 total - 5 per khatabook)

Common items:
- Rice (1kg)
- Wheat Flour (5kg)
- Sugar (1kg)
- Tea Powder (250g)
- Cooking Oil (1L)
- Pulses (500g)
- Soap Bar
- Toothpaste
- Shampoo Pouch
- Biscuit Pack
- And more...

**Details:**
- SKU: 8-character alphanumeric
- Barcode: 13-digit numeric
- Purchase Price: ₹50 to ₹500
- Selling Price: 20-80% markup over purchase
- Current Stock: 10-100 pieces
- Tax Rate: 0%, 5%, 12%, 18%, or 28% (GST rates)
- HSN Code: 8-digit code

### 7. Invoices (8 total - 2 per khatabook)

- **Invoice Number**: INV-0001, INV-0002, etc.
- **Invoice Date**: Last 30 days
- **Due Date**: 30 days from invoice date
- **Status**: 50% SENT, 50% PAID
- **Items**: 2-4 items per invoice
- **GST Invoice**: Yes for business khatabooks

**Invoice Item Details:**
- Linked to inventory items
- Quantity: 1-10 pieces
- Rate: From inventory selling price
- Tax calculated based on item tax rate
- Subtotal, Tax Amount, and Total calculated

**Sample Invoice:**
```
INV-0001
Date: 2026-03-15
Due Date: 2026-04-14
Customer: Amit Gupta

Items:
- Rice (1kg) x 5 @ ₹50 = ₹250 + ₹12.50 tax (5%) = ₹262.50
- Cooking Oil (1L) x 2 @ ₹120 = ₹240 + ₹28.80 tax (12%) = ₹268.80

Subtotal: ₹490.00
Tax: ₹41.30
Total: ₹531.30
```

### 8. Reminder Logs (4 - 1 per khatabook)

- Sent to first customer with positive balance and phone number
- **Type**: WhatsApp
- **Status**: SENT with delivered timestamp
- **Message**: Balance reminder with amount
- **Date**: Last 7 days

### Option 2: Direct Database Query

Connect to your PostgreSQL database and query:

```sql
-- Check data counts
SELECT 'users' as table, COUNT(*) FROM users
UNION ALL
SELECT 'khatabooks', COUNT(*) FROM khatabooks
UNION ALL
SELECT 'customers', COUNT(*) FROM customers
UNION ALL
SELECT 'transactions', COUNT(*) FROM transactions
UNION ALL
SELECT 'invoices', COUNT(*) FROM invoices;

-- Check customer balances
SELECT
  k.name as khatabook,
  c.name as customer,
  c.opening_balance / 100.0 as opening_balance_rs,
  c.current_balance / 100.0 as current_balance_rs
FROM customers c
JOIN khatabooks k ON c.khatabook_id = k.id
ORDER BY c.current_balance DESC;
```

## Balance Calculation Verification

Run this query to verify balances are correct:

```sql
SELECT
  c.name,
  c.opening_balance / 100.0 as opening_rs,
  COALESCE(SUM(CASE WHEN t.type = 'GOT' THEN -t.amount ELSE t.amount END), 0) / 100.0 as transactions_rs,
  c.current_balance / 100.0 as current_rs,
  (c.opening_balance + COALESCE(SUM(CASE WHEN t.type = 'GOT' THEN -t.amount ELSE t.amount END), 0)) / 100.0 as calculated_rs,
  CASE
    WHEN c.current_balance = (c.opening_balance + COALESCE(SUM(CASE WHEN t.type = 'GOT' THEN -t.amount ELSE t.amount END), 0))
    THEN '✓ CORRECT'
    ELSE '✗ MISMATCH'
  END as status
FROM customers c
LEFT JOIN transactions t ON c.id = t.customer_id AND t.deleted_at IS NULL
WHERE c.deleted_at IS NULL
GROUP BY c.id, c.name, c.opening_balance, c.current_balance
ORDER BY c.name;
```

All rows should show `✓ CORRECT` status.

## Resetting the Database

To start fresh:

```bash
cd apps/backend

# WARNING: This deletes all data!
npx prisma migrate reset

# This will:
# 1. Drop the database
# 2. Recreate the database
# 3. Run all migrations
# 4. Run the seed script automatically
```

## Notes

- All monetary values are stored in **paise** (1 rupee = 100 paise)
- Phone numbers use **E.164 format** (+91XXXXXXXXXX)
- Dates use **ISO 8601 format** (YYYY-MM-DDTHH:mm:ss.sssZ)
- All timestamps are in **UTC**
- Soft delete is implemented (deleted_at column) but not used in seed data
- Transaction dates are spread over 90 days for realistic reporting

## Common Issues

### Issue: "relation does not exist"

**Solution**: Run migrations first
```bash
npx prisma migrate dev
```

### Issue: "Cannot find module '@prisma/client'"

**Solution**: Generate Prisma Client
```bash
npx prisma generate
```

### Issue: "Cannot find module '@faker-js/faker'"

**Solution**: Install dependencies
```bash
pnpm install
```

### Issue: "connect ECONNREFUSED" or "database connection failed"

**Solution**: Check DATABASE_URL in .env file and ensure database is running.

## Customization

To modify seed data:

1. Edit `prisma/seed.ts`
2. Adjust counts in the main() function:
   - Change `for (let i = 0; i < 2; i++)` to create more/fewer users
   - Change `faker.number.int({ min: 10, max: 15 })` for customer count
   - Change `faker.number.int({ min: 25, max: 50 })` for transaction count
3. Run the seed again
